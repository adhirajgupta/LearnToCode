Area 1 - Understanding the HTML Document

  • What is HTML / XML / XHTML

HTML stands for HyperText Markup Language. It is tagging language for defining structure. It is the guts that runs the content display of all websites and many of the apps you use on your phone or tablet. HTML is a specific type of XML. XML stands for eXtensible Markup Language, and defines any tagging language that uses <> style tags. The full list of HTML tags can be found here [http://www.w3schools.com/tags/default.asp]. XHTML, which stands for eXtensible HyperText Markup Language, is strict HTML, which means that if you fail to close a tag properly, it will result in an error. If you fail to close a tag properly in regular HTML, it will simply assume that one tag closes where the next begins.

All examples I put will be XHTML compliant, and all of your documents should strive to be as well.

HTML, at its core, is very simple. You can create a document with only the knowledge of 3 tags:

<html>

        <body>

                <p>This is a Paragraph</p>

        </body>

</html>

All HTML documents are wrapped in an <html> tag. The first tag, <html>, is the open tag, and the last tag, </html>, is the close. All closing tags are indicated by a / (forward slash). Everything from the start tag to the end tag is known as an element, or an individual component of an HTML document. The tags indicate that everything between the <html> and </html> should be interpreted as HTML. They also indicate that everything between the <html> and </html> belongs to that tag. The same is true for any other tag. So, the <body> and <p> (paragraph) tags belong to the <html> tag, and the <p> tag belongs to the <body> tag.

A note about indentation:

HTML ignores whitespace (carriage returns, extra spaces beyond the first, tabs, indents, etc), this allows you to format your code in a way that makes it easy to write and edit. In coding, indents (tabs) are used to help keep track of where you are, and what tag you are working in. All of my examples will follow this format, and it is a good habit to have, but it is worth noting, the above example can also be written like so and display exactly the same:

<html><body><p>This is a Paragraph</p></body></html>

  • The document structure

Every HTML page has two sections, a <head> and a <body>. The <head> is optional and stands for header. It is where you can include additional files (such as a CSS file for controlling the look of your document), include a font, include a JavaScript file (for adding interactivity), or define document metadata (such as the title of your document, search keywords, description, etc).

All content you want to display gets placed in the <body> tag.

If you’re new to HTML/XML or tagged content, try to think of HTML as a house. Let’s create our HTML house:

<html>

</html>

A house has an address, and some other non-physical properties - those go in the head. Things in the head tag that hold data are called "meta" (for metadata) tags.

<html>

        <head>

                <meta name="address" value="1 Pleasant Lane"/>

                <meta name="year_built" value="2014"/>

        </head>

</html>

*Note: In the above, the <meta> tags do not have a closing tag. Some tags can be (or must be) self-closing, meaning that they end with /> instead of > which denotes that it is the end of that tag.

Our house also has a structure. It is made up of rooms. To keep things fully HTML, let’s call our rooms divisions or divs (the <div> tag)

<html>

        <head> … </head>

        <body>

                <div>The Living Room</div>

                <div>The Dining Room</div>

                <div>The Bedroom Room</div>

                <div>The Kitchen</div>

                <div>The Bathroom</div>

        </body>

</html>

That’s it.

  • Structure vs. Style Tags

HTML contains quite a few tags -- more than 100, in fact -- but in most cases, your document will only use a handful of different tags (probably 10 or 20). There are two different types of tags, structural and formatting (style). A structural tag implies that the content inside has some significant structure. A formatting or style tag simply changes the way the text is displayed.

In the house example, the <div> tag is a structural tag that denotes the contents are a room. I will demonstrate later how we can label that <div> in a special way to help style it later.

The style tags are a relic of older versions of HTML where there was no CSS to change the way text was displayed. Those tags are <b> (bold), <i> (italic), <strong>, <em> (emphasis), and <center>. While the first 4 tags are still valid, it is recommended that you do not use them as it implies formatting a fixed link between formatting and structure.

An example of why you would not want to use the <b> tag to bold something is as follows:

Let’s say you were setting up a dictionary and wanted to make all the definitions appear bold. You may set up your definitions as such:

<div><b>Happy:</b> delighted, pleased, or glad, as over a particular thing.</div>

If your boss decides that all definitions should be italic instead of bold, you either have to overwrite the default display for <b> tags (which can work, but makes it difficult to look at the source and see what is going on).

By using structure tags (and CSS, discussed in later lessons), we can achieve much smarter results.

Area 2 - Including Additional Files & CSS Part 1

Ordering & Display

I’ve discussed how HTML should house the structure of your content. What about display? The display of your content should be controlled by CSS (Cascading Style Sheets). CSS is a style sheet language that is used to describe the look and format of a document written in HTML and other markup languages. CSS tells the web browser how to display HTML content. CSS can be applied on 4 different levels, in this order:

  1. Directly on the element (should be avoided when writing HTML)
  2. Element ID
  3. Element Class
  4. Tag type

So far, I’ve only showed you the tag type. So, in our first sample document, the <p> tags are a tag type of "p". HTML has 3 additional optional elements that can be added to tags - class, id, and style.

An element with all those tags is below:

<p id="my_first_paragraph" class="no-indent" style="">My first paragraph.</p>

The order of those elements within the tag is not required, but good style says id, class, then style. As noted, you should avoid setting the style directly on the element when it comes to creating your HTML/CSS.

The ordering for CSS is important, as it determines how conflicts are resolved. Let’s say you define all <p> tags to have an indent, but the id of "my_first_paragraph" to have no indents. Since ID is higher up on the list, the <p> will not have indents.

Note about naming

Id and class names cannot have spaces in them, and should be alphanumeric with _ or - in place of spaces. There are many different naming conventions that work for different people. I suggest you name things logically so you can read the code and understand what is being displayed.

  • Using the <HEAD> to add additional files

While you can technically write CSS code directly in your HTML document, it makes things difficult to read, and can make your HTML document very long. Additionally, your CSS should be shared by multiple documents.

You can include a CSS file in your HTML document by using the code:

<link rel="stylesheet" type="text/css" href="style.css"/>

The / at the end of the tag means that it is a self-closing tag (has no contents). It is short-hand notation for:

<link rel="stylesheet" type="text/css" href="style.css"></link>

There are a few tags that HTML will allow you to leave unclosed - the <link>, <br> (line break), and <img> (image) tags are a few examples, but in all my examples, I will include the / to make them self-closing and therefore XHTML compliant.

Next, copy the self-closing tag above into the <head> of your html document. You’ll now need to create a new file and save it in the sample folder as your HTML document. This new file will be named "style.css".

Let us start by defining how the paragraph tag should display. In style.css, input:

p {

        text-indent: 24px;

}

Then save, and load your document in a web browser of your choosing. You should see each paragraph indented 24 pixels (px). Measurements in CSS can be done in many ways (http://www.w3schools.com/cssref/css_units.asp) but I recommend you do them in PX or EM. EM is a relative term for active font size, so a text-indent of 2em would be bigger for a paragraph with larger text, and smaller for one with smaller text, whereas 24px will be the same in both cases.

  • Controlling display with CSS

Basic CSS can be broken down into two sections:

(conditions) {

        (styles)

}

The conditions are a list of tags, ids, and classes. The styles are a list of properties to apply to anything that meets the conditions. In the above example, "p" denotes the <p> tag. The condition is "all p tags" and the style applied is "text-indent: 24px;" Each style has 4 elements.

  • Style Name
  • :
  • Value
  • ;

The style name is what, the colon (:) separates the name from the value, and each statement is ended with a semicolon (;).

As noted before, we can have tags, classes, and IDs in our CSS. To denote a tag, we simply write its name. div for <div> tags, p for <p> tags, etc. For classes, we add a period (.) before the name.

.no-ident {

        text-indent: 0px;

}

says that anything with the class "no-indent" should have a text-indent of 0 pixels. A tag can have multiple classes, separated by spaces.

Example:

<p class="definition no-indent first_on_page">Happy</p>

To represent IDs, you preface the ID with #.

Example:

#my_first_paragraph {

        font-size: 32px;

}

In your conditions, you can combine elements in a boolean fashion. Using a space to separate denotes AND, while using a comma (,) denotes OR. For example:

p, div { … }

says apply the styles to all <p> and <div>

p .no-ident { … }

says apply the styles to all <p> tags that also have the class ".no-indent"

You can combine these as well:

p .no-indent, div { … }

which applies to the styles to all <div> tags and <p> tags that also have the class .no-indent.

For those who like long charts, the full list of css tags and their values can be found here [http://www.w3.org/TR/CSS2/propidx.html]

A quick reference of the most common text-based CSS elements are here:

  • color - defines the color of the text in RGB format, also known as hex code: #RRGGBB where the number is 3 pairs of hex numbers that denote the amount of Red, Green, and Blue. #FFFFFF is white, #000000 is black, #FF0000 is red, etc. Additionally, you can also use the rgba() function to define the color in demical format. This also lets you define an alpha (transparency). While hex notation is universally supported, rgba is less so.
  • text-align - defines the alignment. Values: left, right, center, justify.
  • text-indent - defines the amount of space that should be displayed at the beginning of the element. Values in px (pixels) or em (relative).
  • font-size - defines the size of the font. Values in px or em.
  • font-family - defines which font to use. Recommended you don’t use this for now. Will cover later.
  • font-weight - defines how bold a font should be. Valid values 100 - 900 (400 normal, 600 is bold, 300 light).
  • font-style - defines if a font is italic or not. Values: normal, italic.
  • background-color - defines the background of the element. Values in #RRGGBB. Background color can also be defined in rgba().

  • Understanding Cascading

As a reminder, the Cascading in CSS means that an element will take its highest form value if you enter conflicting information. So, if you had the following CSS:

p {

        font-color: #FF0000;

}

#first_paragraph {

        font-color: #00FF00;

}

and the html:

<p id="first_paragraph">This is first</p>

The font color would be green (#00FF00) because the ID is higher order than the tag. The order is style, id, class, then tag.

Area 3 - CSS Part 2

  • The box model

CSS defines more than just the way text displays. It also defines all of the visual aspects and positioning of every element in an HTML page. All html elements are defined in one of two groups -- inline and block. Inline elements ignore all positioning and special formatting (or, put another way, only respect the text-specific formatting). Block elements respect all CSS.

The inline elements are listed here [https://developer.mozilla.org/en-US/docs/HTML/Inline_elements] but a short list of common ones is below:

  • <b>
  • <i>
  • <span> (like a div, but inline, whereas <div> is block)
  • <a> (used for links)
  • <img>

If you have something like an image that you want to be treated as a block element, you can use the CSS: "display" with value "block" to overwrite the default of inline. Example:

img {

        display: block;

}

You may not specify the height, width, or border for any item that is inline. You need to change its display to inline-block or block to set either of those properties.

Block and the Box Model

All items that are block items fit into the box model [http://www.w3schools.com/css/css_boxmodel.asp]. The box model means that all block elements have properties that apply to their content, padding, border, and margin.

Content

The CSS properties height, width, and background-color affect the content.

Padding

Padding is the space around an object that separates it from text and other objects on the page. The CSS properties padding-left, padding-right, padding-top, and padding-bottom (or just padding if you want a uniform padding) as well as background color affect the area considered the padding.

Border

The CSS property border [http://www.w3schools.com/css/css_border.asp] effects the border.

Margin

The CSS properties margin-left, margin-right, margin-top, and margin-bottom (or just margin if you want a uniform margin) affect the margin. The margin is extra spacing around an element outside of the border.

A note about sizing

If you set your content width to 100px, with a padding of 5px, a border of 1 px, and a margin of 10px, the total width is 132px. Left margin of 10 + left border of 1 + left padding of 5 + 100 content + right padding of 5 + right border of 1 + right margin of 10 = 132. This is a point that can get tricky for many people - as they may set a container to be 600 px wide and put 6 elements in that are each 100 px wide yet only 5 display with the 6th on the last line, forgetting that margin, padding, and border all add to the total width.

Percentages

Font sizing, as I’ve noted, is measured in px (pixels) or ems. When sizing and adding padding/margin, you can use px or ems, and you can also use percentages. Any item with a percentage as a width or height will be a percent of its container -- not the entire document. If the container does not have a set height or width, the item will grow to its container until it meets something that does have a height or width (or it reaches the size of the browser window).

  • Floats, dropcaps, and flair

Using what we know about the box model, we will now go back to our document and add some flair. In writing, we have two decorations that are frequently used - large caps and dropcaps. Large caps make the first letter of the first paragraph large and styled, dropcaps does the same but lines the top of the letter with the top of the rest of the text.

The first thing we need to do is to mark up the HTML. Let’s go back to our first paragraph and wrap the first letter in a span:

<p><span class="large_first">T</span>his is my first paragraph. It goes on and on and on…</p>

You may want to add more text to see how things wrap, but not necessary for the example.

Now, load your HTML file. The span should not be visually noticeable, as we haven’t added any styles to it. First, let’s style it to be a large cap. Edit your style.css file and add a new declaration:

.large_first {

        font-size: 3em;

        font-weight: 600;

}

Save style.css and reload your HTML file. The first letter is now large and bold. You may notice that the line-height for the first line is now larger than the rest. This can be fixed by setting your line height for your p tag. Example:

p {

        line-height: 32px;

}

Now we’ll want to make it a dropcap. To do this, we need to use a block element. To avoid having to change our HTML code, we will do this in CSS:

.large_first {

        font-size: 3em;

        font-weight: 600;

        float: left;

        line-height: 1em;

}

To make the element sink, we’ve applied the css style "float" and told the item to float to the left. Float tells the document to wrap text and other objects around the content. You can float content left or right. Additionally we have to set the line-height (if you did not set the default line height for <p> tags, then you may not need to override this, but it doesn’t hurt to be specific).

If you load this up, you’ll realize that the paragraph text is right next to the first letter. You’ll want to add a little bit of spacing by adding the margin property.

.large_first {

        margin-right: 4px;

        font-size: 3em;

        font-weight: 600;

        float: left;

        line-height: 1em;

}

Like learning a spoken language, you can be considered conversational when you know 5-10% of a language. The same goes for programming. You can build many beautiful documents knowing only a few HTML tags and a few CSS styles. What is most important is that you know how to read documentation. When you want to do something that you are unsure how to do, documentation will be the key.

The <img> tag

There are two ways you can place an image on a page, one is with the <img> tag, and the other is with the css property background-image. For now, I’ll just go into detail of using the <img> tag. As noted above, the <img> tag is a self-closing tag, as well as an inline tag. Let’s start with our basic <img> tag:

<img/>

That tells the HTML document that there is an image, but what image? The next thing we need to add is an attribute named "src" for source.

<img src=""/>

We need to give it an image. We’ll use this happy face: http://upload.wikimedia.org/wikipedia/commons/e/ec/Happy_smiley_face.png

Putting it all together we have:

<img src="http://upload.wikimedia.org/wikipedia/commons/e/ec/Happy_smiley_face.png"/>

Put this into our HTML document and you’ll see a huge smiley face. Since it is pretty big, we’ll want to size the image. We’ll use an ID.

<img id="smiley_face" src="http://upload.wikimedia.org/wikipedia/commons/e/ec/Happy_smiley_face.png"/>

Now let’s edit our style.css document and add the lines:

#smiley_face {

        width: 150px;

}

A note about image sizing

An image will keep the same proportions if you give it only one dimension. So if you only give it a height or a width, it will simply shrink or grow. If you give it both a height and a width, it will distort to fit those exact sizes.

Area 4 - Pseudo Classes & states

CSS can do some extremely powerful things. While much of that can be done using direct tags, classes, etc, there is a second type of class called a pseudo-class which is derived from a state, or ordering.

The full documentation on pseudo can be found at w3 [http://www.w3.org/TR/CSS2/selector.html].

For demonstration and to get you started, I’m going to focus on the :hover, :active and :first-child pseudo classes. All pseudo classes start with a colon (:).

:hover

The :hover pseudo class defines the styles for when your mouse or pointer is hovering over the element. Note: Content that appears when you are hovering will not appear on touchscreen devices.

:active

The :active pseudo class defines the styles for when an item has been clicked/touched. Unlike :hover, this works on touch screen devices.

:first-child

The :first-child pseudo class applies to the first element in a collection.

The best way to see what these do is to see them in action. Edit your style.css document and add the following:

p:hover {

        color: #00FF00;

}

p:active {

color: #0000FF;

font-weight: 600;

}

Save, then reload your HTML document. Move your mouse over any paragraph, and you’ll see the text color change. Click the paragraph and it will change again and become bold. You can change any css property in these pseudo classes.

Using :first-child is a helpful form of shorthand because it saves you from having to add classes to your document for items that are first. It also allows you to quickly move content around in your document and ensure that things will display correctly. To give a visual, let’s add the following to our style.css:

p:first-child {

        font-style: italic;

}

The first paragraph on the page will appear italic, while the rest will remain normal. Where first-child gets more powerful is when combined with the > condition. > means "direct child of".

For example, take the following HTML:

<div>

        <p>...</p>

        <p><span>...</span></p>

        <p>...</p>

        <p>...</p>

        <p>...</p>

</div>

<div>

        <p>...</p>

        <p><span>...</span></p>

        <p>...</p>

        <p>...</p>

        <p>...</p>

</div>

If we wanted to apply our styles to the first <p> tag in each division (<div>) then we’d use: div > p:first-child { … }

Area 5 - An Introduction to CSS3 & Media Queries

CSS3 is the latest revision of CSS. The CSS2 spec focused mainly on the display of content. Originally, CSS2 dealt in static display and focused on the simple and logical display of content. CSS3 focuses on animation, complex layouts, vertical text, and other, more complex elements. Its support within e-readers is limited (pretty much only iBooks and web-based readers accept any amount of CSS3). Even amongst modern browsers, support is limited. That said, it’s the future of CSS. It does some wonderful things, and many of its features are fairly universal.

  • What works in CSS3

One of the best resources in HTML/CSS development is "Can I Use" : http://caniuse.com/

It lists the compatibility of every HTML and CSS feature in all browsers. While it doesn’t cover reading systems, you can usually find a good substitute.

Below is a list of some CSS3 properties that are well-supported and heavily used:

  • border-radius: This creates a radius around the corners of the box model. This rounds the border itself, while padding and margins are left untouched.
  • transition: This defines a set of parameters to determine a smooth animation when changing an attribute value. For example, you can set a transition of 1s, and if you change the width of an item, it will grow over the span of 1 second.
  • animations/keyframes: This allows you to create a list of properties based on a timeline that an element will cycle through. That way, your element will transition from one state to the next.
  • Transform: Transform lets you rotate an object along a three-dimensional plane. CSS2 was based on two-dimensional x and y coordinates with a fake z axis that really was just a layer (what items will display on top of other items). Transform lets you skew elements in ways you could not before.

A full list of CSS3 properties can be found here: http://www.quackit.com/css/css3/properties/.

CSS3 is very useful, especially when paired with JavaScript. 

  • What is to come in CSS3

Column, vertical writing, and many other properties are to come. For a full list of working groups and statuses, you can check out the w3c css page [http://www.w3.org/Style/CSS/current-work.en.html].

CSS3 allows you to decouple your display and content structure even more than in the past. It also allows you to define a really interesting set of user interface patterns.

Media Queries

CSS allows you to create responsive layouts using something called a media query. Media queries can be used in one of two ways. One is to conditionally load a CSS file, the other is to conditionally load specific CSS within a css file.

The most often-used queries are:

  • max-width: Any display with this width or less.
  • min-width: Any display with at least this much width.
  • max-height: Any display with this height or less.
  • min-height: Any display with at least this much height.
  • orientation: Landscape or portrait.

You can see full documentation on media queries here: [https://developer.mozilla.org/en-US/docs/Web/Guide/CSS/Media_queries]

To conditionally load a CSS file, use this format:

<link rel="stylesheet" media="(max-width: 800px)" href="example.css" />

To conditionally load CSS within a CSS file, you use the media query as such:

@media (max-width: 800px) {

p {

font-size: 16px;

}

}

While media queries are often used to alter the display between devices (and therefore format the same HTML optimally for phones, tablets, and desktops), you can test media queries simply by resizing your browser window.

Area 6 - JavaScript - what and how

  • Interactivity & Frameworks

HTML and CSS are static, meaning that what you write is what you get. Even with CSS’s state-based pseudo-classes, the ability to have interactive elements, complex animations, or anything mutable is drastically limited. To add interactivity, JavaScript was introduced. JavaScript is an object-oriented programming language that allows you to directly manipulate the HTML document (also known as the DOM or Document Object Model). JavaScript can be used as a stand-alone language to perform tasks and do computation, but I will focus on its use within the context of an HTML/CSS website.

Note about ebooks

As of this course, the only reading system that supports JavaScript is iBooks and web-based e-reading systems (and even some of the web-based systems disable JavaScript). Nonetheless, it used in nearly every website you visit and is the core of interactivity. A basic knowledge of JavaScript will arm you with the ability to create amazing and wonderful things.

JavaScript Frameworks & Existing Code

Web-based JavaScript has an extremely plentiful amount of open source frameworks and plugins. These are pre-written sets of code that are meant to make complex tasks easier, or to enable you to add complex interactivity very simply. While the code behind many of these frameworks and widgets is extremely complex, they are created in such a way that they are very easy to include in your document.

The most widely-used framework is jQuery (http://jquery.com/). jQuery serves a few purposes:

  • It handles compatibility really well. Functions that are normally only available to more modern browsers get special support within jQuery so that your code will work in the largest amount of web browsers.
  • It makes things a little bit easier to do. Its original goal -- and why the "query" is in its name -- was to make it really simple to find specific elements on the page and modify their properties. What used to require many lines of JavaScript code can be done in 1 line of jQuery that is much easier to read.
  • It allows for very simple plugins. There is a very large development community that build jQuery plugins. There is an official repository here (http://plugins.jquery.com/) but a google search for "jquery plugin [what you want to do]" will yield many additional results. These plugins simply require you to include a JavaScript file in your page and write only a few lines of code. We will explore using a slideshow widget later in this lesson.

  • The absolute basics

JavaScript, like CSS, can be inserted into multiple places within your document. The recommended usage is to include all of your JavaScript in an external file (like your CSS). If you need to include JavaScript within an HTML document, it needs to be included within a <script> tag.

Let’s start by creating and including a JavaScript file. Under the line where you included jQuery to your HTML document add the code:

<script src="script.js"></script>

Next, create a new text document and save it as "script.js" wherever you have your index.html saved. Because we are including this file in a <script> tag, we do not need to duplicate the script tag within the script.js file.

Let us start by writing our first line of JavaScript (well, for some of you your first line.). Add the following:

alert("Hello World!");

Why Hello World?

It is a programming thing. The first program you should write should tell you “Hello, world!” It’s tradition.

Save the file and reload your index.html. It should pop up a window saying "Hello World!"

What does object-oriented mean?

I mentioned above that JavaScript is object-oriented. That means that everything within JavaScript that you deal with is an object. Each object has a type, which carries along with it a set of properties and functions. A real-world example is a car. A car would be a type of object and every car would have a gas pedal that, when pressed, makes the car go. A car also has a brake pedal that, when pressed, makes the car stop, and a steering wheel that turns the car. And so on. Every instance of a car has the same functions (although you can change the properties of each instance of a car, such as its color, make, number of doors, etc).

To use any functions of an object you use the period (.) followed by the function you want to use.

For example, if we take a string and want to replace its contents we can use:

alert("JavaScript is OK".replace("OK","Great!"));

If you run that, it will alert you "JavaScript is Great!"

In this instance, the ".replace()" function is like the brake. Just as the brake enables you to stop in every car, the .replace() function enables you to replace properties with other properties. In every string of JavaScript, this function does the same thing to every string object.

You may have noticed that every line of JavaScript (instruction) is ended with a semicolon (;). While it is not always necessary to do this, I highly recommend it.

Functions

Along with having objects, JavaScript also has functions. A function takes input and provides output. A function can technically take no inputs, and could provide no output (but then it wouldn’t really be very useful.). The notation for writing a function is to use the word function. For example:

function makeMeHappy(str) {

        return str.replace("sad","happy");

}

In this function, we are taking the input of "str" and returning that string with "sad" replaced with happy. To call this function, we use the line:

alert(makeMeHappy("I am very sad"));

which will alert "I am very happy".

Notes on naming conventions:

You can name your objects, variables, and functions anything you wish. (Well, almost. There are special characters you have to avoid using, and you cannot start with a number.) But there are a few conventions that are used to make code easier to read and understand.

Variables - All lowercase, usually broken up with underscores ("_"). Variables should be descriptive, for example: "number_of_clicks" and "name_of_user".

Functions - Lowercase first letter, capitalize the words after. Example: makeMeHappy, removeDropCaps, makeTextBlack, etc. Functions do something, so their names should be descriptive action words. This will make your code very easy to read and understand.

Objects - Capitalize every word, no underscores. An object is a thing, so they should be named logically. For example: User, Box, GameBoard, etc. Until you get into more complex JavaScript programming you will not need to be creating objects, you will simply be using existing objects.

  • The Document Object Model (DOM)

The DOM, or Document Object Model, is the JavaScript representation of the HTML and CSS that you see on the page. Changing anything to the DOM will update the HTML on the page to match accordingly.

In jQuery, you select elements very similarly to the way you would in CSS. It is important to note that any changes to CSS using JavaScript happen at the actual item level, so it will overwrite any prior CSS you may have (the item is the highest level). JavaScript cannot be used to edit the actual CSS.

Let’s start by using jQuery to select all paragraphs and make their font grey (#CCCCCC):

$("p").css("color","#CCCCCC");

Add that line to your JavaScript file (remove the alert statement), then save. When you reload your index.html, it will display all the paragraph text in grey.

Let’s break the statement down. Any jQuery statement starts with $. Then it is followed by the query, which is within quotations (""). We wanted to select our <p> tag, so, like in CSS, we simply used "p". We could also select classes and ids like this:

$(".smiley_face").css("width","100px"); // for class "smiley_face" make 100 px wide

$("#first_paragraph").css("font-size","2em"); // make id first_paragraph 2em font-size

Comments

In JavaScript, you can insert comments (non-interpreted code) by using // or /* */. Anything that follows // is ignored by JavaScript. Anything between the ** within /* */ is also ignored. Comments are used to help you remember what it is you are trying to do, and to make it easier for others to read your code. Examples of good comments:

//Create the variables for saving state

/* This function will take in a string and replace "sad" with "happy" */

Comments with // must be on the same line. Comments using /* */ can span multiple lines.

  • The event stack/Interactivity

Simply changing CSS via JavaScript isn’t terribly interesting - as we could just set the CSS and be done. Where things get interactive is when you start working with events. For simplicity, we’ll focus on the events that jQuery offers (which may be a complete list of events, but some browsers may have other odd unique events). You can see a full list of events here [http://api.jquery.com/category/events/] but I’ll be discussing the following events:

  • .change() - Triggers when the value of an element changes (applies only to form elements, so we won’t cover it much until later).
  • .click() - Triggers when an object is clicked or tapped (on a touch screen).
  • .dblclick() - Triggers when an object is double-clicked or double-tapped.
  • .hover() - Triggers when you move your mouse over an object (does not trigger for touch screens).

The rest of the events are very useful for doing complex interactions, but for most basic things, the four above will serve perfectly.

Waiting for the document to be ready

JavaScript loads in the order the scripts are included. Additionally, some scripts take longer to load than others, and could possibly run before your document has fully loaded. To prevent any sort of conflict, jQuery has added an amazingly wonderful function:

$("document").ready(function() {

        //CODE GOES IN HERE

});

What this function does is delay the execution of any code within until everything is loaded and ready to go. Most -- if not all -- of your JavaScript will go here (the only thing outside of here would be declaring variables or functions).

Let us start by clearing out the contents of our script.js file and adding the following:

$("document").ready(function() {

        $("p").click(function() {

                $(this).remove();

        });

});

Let me first walk you through the code. We have our $("document").ready which I already discussed. The next item we have is our selector: $("p"). This means everything that follows applies to all of our paragraph tags. Following the selector, we have .click() which means that we are setting the functionality for a click action.

The functionality is going to be a function, so we use function(). All the code for that function is going to go between { and }. The code for that function in "$(this).remove();"

Let’s talk about the 2nd part. The .remove(); is a jquery function that deletes the DOM object (and the HTML) from the visible document. That means the object will simply disappear. The first part is $(this). The $() is required to say that something is a jQuery object (and we can use the jQuery functions on it). this is a reserved keyword to refer to the object that is being acted on.

Since there are multiple <p> tags in our document, we don’t want to remove all of them, just the one that is being clicked. We can do that using this.

Save your script.js file and reload your index.html. Now, try to click on any paragraph. When you do so, it will delete. Don’t worry. This is temporary. Changes to the DOM do not actually save to the file, they are display only. If you want all of your content back, just reload the page.

 

Removing items is great, but most likely it is not what you want to do. For the next example, we’re going to create a font size changer. It will require us to create two buttons in HTML - one to increase the font size, one to reduce the font size.

Adding the HTML

We’ll need to edit our index.html and add our two buttons. While you could technically use a div, span, or any other element, we’ll use the <button> element.

Add this to your index.html at the bottom before the </body> tag:

<button id="increase_font_size">+</button>

<button id="decrease_font_size">-</button>

Add this to your style.css file

#increase_font_size {

        position: fixed;

        top: 8px;

        left: 8px;

        height: 32px;

        width: 32px;

}

#decrease_font_size {

        position: fixed;

        top: 46px;

        left: 8px;

        height: 32px;

        width: 32px;

}

Absolute Positioning

If you want to specify the display location of an item, you can tell it to position the item in a fixed place by using position: fixed;. Then, you define a top or bottom, left or right (which will put the items left, top, bottom, or right at that position on the page). You can define position in px or %. You can also position something absolutely, using position: absolute; which will place it on top of other elements. The difference between fixed and absolute is that absolute will scroll with the document, while fixed will remain in place even if the document scrolls.

This CSS will position the items at the top left of the page.

Remove the old code from your script.js file and add this:

var current_font_size = 16;

$("document").ready(function() {

        $("#increase_font_size").click(function() {

                current_font_size = current_font_size + 2;

                $("p").css("font-size",current_font_size+"px");

        });

        $("#decrease_font_size").click(function() {

                current_font_size = current_font_size - 2;

                $("p").css("font-size",current_font_size+"px");

        });

        $("p").css("font-size",current_font_size+"px");

});

Save your script.js file and reload your index.html. Click on the + or - button to see the font size of all your paragraphs change.

What is the code doing?

First let’s look at the first line: var current_font_size = 16;  In JavaScript, if you want to store any value, you need to use a variable. JavaScript requires you initialize a variable by declaring it with "var". So, the statement is var [name of your variable] = [something].

In this case, we named our variable "current_font_size" (descriptive and tells us what is stored in it). The value after the = is an object. JavaScript makes an object out of everything. (There is a concept of a primitive, for which numbers can fall, but for simplicity sake, I won’t get into them.) 16 is a Number object. So, I can take that number and perform a function on it, for example: 16.toFixed(2); which will display the number as "16.00" (fixed decimal places).

Now that we have our variable initialized and it has a value, we can use it and modify it later.

Next, we have our jQuery code - which is all inside the $("document").ready() function.

We have 2 click actions that essentially do the same thing, so I’ll just talk about one, as it applies to both.

$("#increase_font_size").click(function() {

                current_font_size = current_font_size + 2;

                $("p").css("font-size",current_font_size+"px");

        });

The first thing I’ll note is the tab indenting. With JavaScript, instead of tags <> we have { and } which denote "ownership" (blocks of code). To keep track, I’ve indented every time I have an open { and reduce every time I have a }. This make the code easy to see what belongs where.

We have our selector: $("#increase_font_size"). This selects the HTML element with the ID of "increase_font_size" which is our button that we want to increase the font size. Now let’s look inside the function. Next, we are going to change the variable current_font_size. The new value we want is the old value + 2. So, we have: current_font_size = current_font_size + 2; There is actually a secondary notation we could use, which is +=. We could write this as: current_font_size += 2; Both statements would give you the same result, but I find the first is easier for beginners to read.

So we’ve changed our variable. Now we need to change our paragraphs. We first select all the p tags: $("p") and modify their css: .css("");

We want to modify the property "font-size" and we want to set the value to our current font size. Because the font size value needs to be in pixels or ems, we have to specify. We do that by adding the string "px" to the result, which gives us:

$("p").css("font-size",current_font_size+"px");

If we were to leave out the +"px" then the browser wouldn’t know the dimensions and things would not display as expected.

The decrease_font_size is the same, but with a -. Again, we could use -= if we wanted, to reduce the value, but both are fine.

Getting current values

If you want to determine a css value instead of set it, you only put in the first parameter. So, if you wanted to get the font size, instead of set it you would do:

$("p").css("font-size");

In the above case, there are many paragraphs, so the result would actually be an array (list) of values.

An IF statement

The most powerful statement in programming is the IF THEN ELSE statement. It allows you to make a determination and do different things based on it. It is what adds intelligence to applications. In JavaScript, the notation of an if statement is very simple:

if (condition) {

        //Run this code if the condition is true

} else {

        //run this code if the condition is false

}

The else { } section is optional.

The condition must be something that equates to a true or false. For example:

if (current_font_size < 10) //if the font size is less than 10

is fine whereas:

if ($("p").css("font-size")) //this is not good

Using the “if” to prevent crazy

Let’s use an “if” statement to prevent the user from increasing or decreasing the font to be too large or too small. Let’s start with too large.

Originally we had

$("#increase_font_size").click(function() {

        current_font_size = current_font_size + 2;

        $("p").css("font-size",current_font_size+"px");

});

We will add our “if” statement:

$("#increase_font_size").click(function() {

        if (current_font_size <= 26) {

        current_font_size = current_font_size + 2;

                $("p").css("font-size",current_font_size+"px");

}

});

As you can see we had to indent the code a little more. Secondly, we have our statement, if (current_font_size < 26) which prevents the code from executing if the font size is 26 or more.

Let’s say we want to alert the user why things aren’t getting bigger, we can use else for that:

$("#increase_font_size").click(function() {

        if (current_font_size <= 26) {

        current_font_size = current_font_size + 2;

                $("p").css("font-size",current_font_size+"px");

} else {

        alert("Cannot increase the font size any more!");

}

});

Area 8 - Putting it all together

  • Using HTML/CSS and JavaScript to create and manipulate a document

You’ve just learned years worth of HTML/CSS/JavaScript in only a few weeks. Well - in reality, I’ve equipped you with the core basics and the tools to grow and learn HTML/CSS/JavaScript. If you feel like things have gone fast - they have. This course is not meant to make you an expert, but to give you the confidence to learn how to find a solution. Ultimately there are a few concepts that you need to memorize to become fluent in HTML, CSS, and JavaScript.

1) You don’t need to be XHTML strict, but you should strive for it.

Every HTML tag should have an open and a close tag. If it is a self-closing tag, don’t forget the closing />. Beyond being good form (and ensuring that your HTML will work perfectly in all ereader systems that require XHTML strict), it also makes it really easy to format your code using tabs which will make it very easy to read.

2) Compartmentalize

HTML allows you to do many things that can form bad habits. Keep your JavaScript, your CSS, and your HTML separate. This will allow you to easily take your CSS or JavaScript to other projects. Additionally, as your code begins to grow, you can take out generic elements and move them to different files. You can include as many JavaScript or CSS files as you want, so have a blast.

Additionally, your HTML should be grouped logically around your content. Don’t worry about display, all of that can be controlled in CSS.

3) Understand the Cascading Order for CSS.

Item, ID, Class, Tag. Repeat: Item, ID, Class, Tag.

As things get more specific, that is how they override. There is one exception to this. You can add "!important" after a value and it will override even more specific css. For example:

p {

color: #000000 !important;

}

The above will ensure that all paragraph text is black, no matter what CSS color change may be in class, id, or on the tag.

4) JavaScript changes an element’s style only.

When using JavaScript to change an element, you can only change the element’s CSS. You can use jQuery to select all items with a specific class, but you are adding CSS to the object itself - which means it will override any previous styling that it has.

5) Chances are, someone else has solved your problem.

One of my most-viewed websites is http://stack-overflow.com It is a developer Q&A site where people ask and answer technical questions. You’ll find answers to everything from simple questions like "How do I add a jQuery plugin to my page?" to extremely advanced questions like "How do I modify the signature of a function to make it mutable?" Documentation exists all over the place.

For simple things, you can check out http://w3schools.com. (Be careful when it comes to advanced features. They tend to have less-than-useful information for complex items.)

6) JavaScript is deep, CSS is shallow, and HTML should be flat.

HTML has a limited number of tags. If your tags match and you are properly using classes and IDs to identify items and classify them, then you’ve got HTML down well. There are advanced uses for HTML, especially when you get into HTML5 canvas, but even then, the complexity comes in JavaScript).

CSS has a wide variety of properties, all of which control some aspect of display. Knowing CSS doesn’t turn you into a user interface expert or a design expert, but it gives you the tools to implement tons of different complex and beautiful layouts. If you know just a few CSS properties, and where to look for documentation, then you’re set.

JavaScript is a real programming language. A lot can be accomplished with just the ability to copy/paste and modify a few values, but it has the potential to completely break as well. For example, if you fail to put a semicolon (;) at the end of a line or enter a bad command, your JavaScript with throw an error to the console and fail to execute any code. Debugging can be quite a pain, but thankfully there are troubleshooting tools available, such as Firebug [https://addons.mozilla.org/en-US/firefox/addon/firebug/] for Firefox.
